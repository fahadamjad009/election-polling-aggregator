# CLAUDE.md

## Operating rule
Before starting work, inspect relevant skills in .claude/skills/.
Use: claim-verification-gate before any status/metric claim,
master-standard-review-gate before declaring anything done,
test-suite-scaffolding when adding tests to an untested repo,
leakage-safe-validation for any split/join/holdout work,
dual-app-deployment-checklist before any deploy claim,
powershell-windows-gotchas for Windows-origin failures.

Do not load unrelated skills. Every completion must name the
verification command actually run.
